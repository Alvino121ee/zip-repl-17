---
name: Multi-persona council debate design
description: How the Dewan Emas (XAUUSD) and Dewan BTC council/team features generate 15-persona debates cheaply.
---

For a "team of N personas debate + 1 leader decides" feature, do not make N separate LLM calls per cycle (too costly/slow). Instead, send one structured prompt per cycle that lists the fixed ordered roster (name + role for all N members) and requires the model to return opinions for all of them plus a leader synthesis in one JSON payload, in the same order given.

**Why:** N personas would mean N LLM calls per cycle × multiple assets × frequent cycles — cost/latency scale badly, and per-persona calls have no shared context so debate coherence suffers anyway.

**How to apply:** Keep the roster order fixed in code (not regenerated by the model) so the response array can be zipped back to member name/role reliably by index. If JSON parsing fails, fall back to a majority-vote-based default per member rather than crashing the cycle — debate cycles run unattended in the background.
