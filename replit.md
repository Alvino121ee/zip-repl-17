# SahamRadar AI

AI-powered analysis and stock-picking tool for the Indonesian stock exchange (IHSG). It scores stocks on trend/momentum/volume/risk, surfaces top opportunities, and runs a simulated "Daily Picks" engine that auto-selects the highest-upside stocks each day and tracks their simulated profit/loss over time.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` ‚Äî run the API server (port 8080, path `/api`)
- `pnpm --filter @workspace/saham-radar run dev` ‚Äî run the web frontend (path `/`)
- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm run build` ‚Äî typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` ‚Äî push DB schema changes (dev only)
- `cd artifacts/api-server && ../../scripts/node_modules/.bin/tsx src/scripts/seed.ts` ‚Äî seed/reseed the simulated stock universe and price history
- Required env: `DATABASE_URL` ‚Äî Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 (artifact: `artifacts/api-server`)
- Frontend: React + Vite (artifact: `artifacts/saham-radar`)
- DB: PostgreSQL + Drizzle ORM (`lib/db`)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec in `lib/api-spec/openapi.yaml`)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/db` ‚Äî Drizzle schema, including `stocksTable`, `stockScoresTable`, and `dailyPicksTable`
- `lib/api-spec/openapi.yaml` ‚Äî source of truth for the API contract; run codegen after editing
- `artifacts/api-server/src/lib/picks.ts` ‚Äî Daily Picks engine (candidate selection, closing stale picks, generating picks for a date)
- `artifacts/api-server/src/routes/picks.ts` ‚Äî Daily Picks REST endpoints (`/api/picks/generate`, `/today`, `/history`, `/report/:date`)
- `artifacts/api-server/src/scripts/seed.ts` ‚Äî seeds the simulated IHSG universe (58 tickers) with fundamentals, price history, and scores
- `artifacts/saham-radar/src/pages/picks.tsx` ‚Äî Daily Picks UI (today's picks, summary cards, history, per-date report)

## Architecture decisions

- **No live market data feed.** There is no real-time IHSG data integration. The stock universe (58 tickers), prices, and fundamentals are simulated/seeded data generated by `seed.ts`, not live feeds. This was an explicit tradeoff communicated to the user ‚Äî full live coverage of all ~900 IHSG-listed stocks needs a real market-data API later.
- **Daily Picks is a simulation, not real trading.** Each pick uses a fixed simulated investment of Rp 10,000,000 per stock to compute simulated profit/loss; no real money or brokerage integration exists.
- **Staleness must be measured against "today", not the generation target date.** `closeStalePicks()` in `picks.ts` closes open picks dated strictly before *today* (`todayStr()`), regardless of what `targetDate` a `generate` call was given. Comparing against `targetDate` instead of "today" is a bug: backfilling/generating picks for a past date would incorrectly close today's still-open picks. See `.agents/memory/daily-picks-staleness.md`.

## Product

- **Dashboard / Screener** ‚Äî browse and filter all 58 simulated IHSG stocks by score, sector, trend/momentum/volume/risk.
- **Daily Picks AI** (`/picks`) ‚Äî each day, the engine auto-selects the top simulated-upside stocks (default 5/day), shows entry price, AI-generated reasoning, and live status (open/closed). A history table tracks cumulative simulated profit across days, and a per-date report shows full pick details for any past day.
- **Watchlist, Compare, Risk Radar** ‚Äî existing stock-comparison and watchlist tooling.

## User preferences

- User is non-technical and communicates in Indonesian; keep explanations in plain, non-technical language.

## Gotchas

- Always restart the `artifacts/api-server: API Server` and `artifacts/saham-radar: web` workflows (not generic names like "API Server") after server-side changes ‚Äî these are the actual registered workflow names in this project.
- No `tsx`/`ts-node` at the workspace root; use `scripts/node_modules/.bin/tsx` to run one-off TS scripts (e.g. seeding).
- After editing `lib/api-spec/openapi.yaml`, always rerun `pnpm --filter @workspace/api-spec run codegen` before using new endpoints from the frontend.
